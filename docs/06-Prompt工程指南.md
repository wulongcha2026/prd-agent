## 1. Agent 工作流 Prompt 架构

```
输入 → 意图识别 → 信息补全 → 需求分析 → PRD生成 → 审核优化
```

每个节点都需要精心设计的 Prompt。

---

## 2. 核心 Prompt 模板

### 2.1 意图识别 Prompt

```
你是一个资深需求分析师。请分析以下需求，确定需求类型。

## 需求类型定义
- product：新产品功能开发
- bug：问题缺陷反馈（用户报告的问题）
- optimize：性能或体验优化需求
- feature_request：功能增强请求

## 任务
分析用户输入的需求，返回最合适的类型。

用户输入：{input}

## 要求
- 只返回类型名称，不要其他内容
- 如果无法判断，返回 "product"
```

### 2.2 信息补全 Prompt

```
你是一个需求分析师。请分析以下需求，识别缺失的关键信息。

## 需求信息清单
完整的需求应该包含：
1. 用户角色 - 谁来使用这个功能
2. 用户目标 - 用户想达成什么
3. 业务价值 - 为什么要做
4. 使用场景 - 在什么情况下使用
5. 验收标准 - 怎么算完成

## 当前需求
需求类型：{intent}
需求描述：{raw_input}

## 任务
1. 分析当前需求缺少哪些信息
2. 返回缺失信息的列表

## 输出格式
返回 JSON 数组格式：
["缺失信息1", "缺失信息2", ...]
如果信息完整，返回：[]
```

### 2.3 追问引导 Prompt

```
你是一个友好的 AI 助手。用户提交了一个需求，但需要补充一些信息才能更好地帮助他。

## 用户原始需求
{raw_input}

## 缺失的信息
{missing_info}

## 任务
请生成一条友好的追问消息，引导用户补充信息。

## 要求
- 语气友好、专业
- 每次只问 1-2 个问题
- 解释为什么需要这些信息
- 提供回答示例

## 输出格式
直接输出追问消息，不要其他内容。
```

### 2.4 需求结构化 Prompt

```
你是一个资深产品经理。请从以下需求中提取关键要素，结构化呈现。

## 需求描述
{raw_input}

## 需求类型
{intent}

## 任务
提取以下要素：

1. 用户角色（User Persona）
   - 谁会使用这个功能？

2. 用户目标（User Goal）
   - 用户想达成什么？

3. 业务价值（Business Value）
   - 为什么要做？有什么价值？

4. 使用场景（User Scenarios）
   - 在什么情况下使用？

5. 验收标准（Acceptance Criteria）
   - 怎么算成功？

## 输出格式
```json
{
  "user_persona": "...",
  "user_goal": "...",
  "business_value": "...",
  "scenarios": ["...", "..."],
  "acceptance_criteria": ["...", "..."]
}
```
```

### 2.5 PRD 生成 Prompt

```
你是一个资深产品经理。请根据以下需求信息生成标准的产品需求文档（PRD）。

## 输入信息
- 需求类型：{intent}
- 需求描述：{raw_input}
- 提取的要素：{requirements}
- 追问获取的补充信息：{additional_info}

## PRD 模板结构

```markdown
# [产品名称] 产品需求文档

## 1. 需求概述
### 1.1 需求背景
### 1.2 需求目标
### 1.3 适用范围

## 2. 用户故事
作为[角色]，我希望[功能]，以便[价值]

## 3. 功能需求
### 3.1 功能点 1
#### 功能描述
#### 输入
#### 输出
#### 处理流程

### 3.2 功能点 2
...

## 4. 非功能需求
### 4.1 性能要求
### 4.2 安全要求
### 4.3 兼容性要求

## 5. 验收标准
| 序号 | 验收点 | 验收条件 | 优先级 |
|------|--------|----------|--------|
| 1 |  |  | P0 |
| 2 |  |  | P1 |

## 6. 依赖关系
### 6.1 功能依赖
### 6.2 技术依赖
### 6.3 外部依赖
```

## 要求
- 内容完整、专业
- 适合开发团队阅读
- 验收标准必须可测试
- 使用中文输出
```

### 2.6 审核优化 Prompt

```
你是一个 PRD 审核专家。请审核以下 PRD 文档，提出优化建议。

## PRD 内容
{prd_content}

## 审核维度
1. 完整性 - 是否包含所有必要章节
2. 清晰度 - 描述是否清晰无歧义
3. 可行性 - 技术实现是否可行
4. 验收标准 - 是否可测试

## 输出格式
```json
{
  "score": 85,
  "issues": [
    {
      "type": "completeness",
      "description": "缺少非功能需求章节",
      "suggestion": "补充性能、安全等要求"
    }
  ],
  "passed": true
}
```
```

## 3. Agent 状态管理

### 3.1 状态定义

```python
class PRDState(TypedDict):
    raw_input: str                    # 用户原始输入
    intent: str                       # 识别的需求类型
    missing_info: list[str]           # 缺失信息列表
    additional_info: dict             # 追问获取的补充信息
    requirements: dict                # 结构化的需求要素
    prd_content: str                  # 生成的 PRD
    review_result: dict               # 审核结果
    iterations: int                   # 追问次数
```

### 3.2 状态流转

```
START
  │
  ▼
intent_recognition (意图识别)
  │
  ▼
information_gap (检查缺失)
  │
  ├─▶ 有缺失 + 次数<3 ──▶ ask_more (追问)
  │                              │
  │                              ▼
  │                        information_gap
  │                              │
  └─▶ 无缺失 ──────────────────▶
                                    ▼
                            requirement_extraction
                                    │
                                    ▼
                            generate_prd
                                    │
                                    ▼
                            review_prd
                                    │
                                    ▼
                              END
```

## 4. 调优参数

### 4.1 LLM 参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| temperature | 0.5 | 适中，保证准确性 |
| max_tokens | 3000 | PRD 较长，需要足够 |
| top_p | 0.95 | 与 temperature 配合 |

### 4.2 检索参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| max_iterations | 3 | 最多追问 3 轮 |
| timeout | 60s | 单次调用超时 |

## 5. 常见问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 意图识别错误 | Prompt 不够清晰 | 补充示例（Few-shot） |
| PRD 不完整 | 提取要素不充分 | 优化结构化 Prompt |
| 追问太机械 | 追问 Prompt 友好度不足 | 增加角色设定 |
| 输出格式不对 | 输出未约束 | 使用 Pydantic 校验 |
